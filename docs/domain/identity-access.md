# Dominio: Identidad y Acceso

## Reglas de negocio

- Las cuentas humanas requieren correo único y recuperación por correo.
- Alumnos pueden existir sin cuenta de portal; padres registrados siempre tienen cuenta.
- Una persona puede tener varios roles sin duplicar su cuenta.
- Los vínculos padre-alumno son N:M y solo los gestiona personal autorizado.
- Las cuentas se desactivan, no se eliminan cuando poseen historial.
- Los permisos sensibles se asignan a cuentas concretas.

## Casos de uso compartidos

- Crear, actualizar, activar y desactivar cuenta.
- Restablecer contraseña.
- Asignar o retirar rol operativo.
- Delegar gestión de usuarios.
- Vincular o desvincular padre y alumno.
- Seleccionar contexto de portal para una cuenta con varios roles.
